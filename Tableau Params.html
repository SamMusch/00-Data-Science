                          Before we begin, here's a quick look at parameters. (Tableau)

                          A parameter in Tableau is the same thing as a variable in programming. They can be used to:

                          • Replace a constant value in a calculation, filter, or reference line.
                          • Automatically refresh the current value, list of values, or range of values

                           

                          Swap Metric

                          This section shows you how to swap out different metrics.

                          • Situation: A developer needs to manually select a metric for each graph. These graphs are then compiled together to create a dashboard. If a business user wants to see other metrics, the developer needs to duplicate the exist graph and swap out metrics.
                          • Problem: For the business user, this is very time consuming. Also, adding too many graphs can easily create confusion. From the technical point of view, duplicated views takes up space and increase the likelihood of something going wrong.
                          • Solution: Use a Parameter instead to swap out available metrics.

                          Parameter

                          The first steps is to create the parameter. These will be the available Metrics to the user.

                          • Values: These need to match the Metric names from over on the left.
                          • Display as: These are the names displayed to the user.

                           

                          Calculated Field

                          Next, we need to create a calculated field. This field will contain whichever Metric is selected.

                          • [Select a Metric:] is the parameter we just created.
                          x
                          1
                          case [Select a Metric:]
                          2
                          when 'Profit' then sum([Profit])
                          3
                          when 'Profit per Order' sum([Profit]) / COUNTD([Order ID])
                          4
                          when 'Shipping Cost' then avg([Shipping Cost])
                          5
                          end

                          And that's it! This is a quick & painless feature to add which makes your viz much more effective.

                           

                          Drill Down

                          This is how to add multiple levels of detail to a graph. For example, the user can start with Sub-Category and then click to see the Products within the hierarchy.

                          • Situation: You are asked to evaluate the performance of sub-categories at your org. There are 10 sub-categories and 50 products within each.

                          • Problem: There is an inherent trade-off between detail & understandibility. If you only show the sub-category level, you could be missing out on meaningful patterns occuring at the product level. On the other hand, displaying at the product level is too overwhelming and makes it tough to draw conclusions.

                          • Solution: Use a Parameter to adjust granularity.

                            Show the 10 sub-categories -->

                            Click on 1 of them -->

                            Show the 50 products within that sub-category

                          Swap Graph

                          Now we are going to see how to swap out graphs to display on a dashboard.

                          • Situation: You are asked to produce a weekly report with a variety of graphs.
                          • Problem: Some users may want to see only a high level of detail, while others may need to dig deeper into the weeds.
                          • Solution: Use a Parameter to swap out graphs. This will let the user select only the info they care about while still keeping the dashboard clean.

                          Dashboard

                          To place these on a dashboard:

                          1. Drag "Horizontal" on the dashboard
                          2. Drag "Map" onto the horizontal section.
                          3. Drag "Scatter" onto the horizontal section. (Like in the image below)
                          4. Click "Hide Title" for each of the 2 graphs. This will consolidate the 2 graphs into 1 section.

                           

                          The final step is to update the parameter action we created in the "Drill Down" section. All we need to do is edit the "Source Sheets" to include the name of the dashboard.

                           

                           

                          Dashboard Interactivity

                          The final task is an easy one. In order to make your dashboard interactive, right-click & select "Apply as Filter" for each graph. As an example, if you were to click on "United States" the rest of the dashboard would contain only US data.
